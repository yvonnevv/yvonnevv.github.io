数据流

保持全站一致，结算页也采用redux作为状态管理器，因为结算页展示数据需要和子页面执行相应操作后的数据保持联动，也就是说存在通信问题，所以笔者将主页面展示的数据储存在redux的全局store中。主页面和子页面共同维护一套数据。来看眼demo。

demo展示的是发票选择页和主页面的数据联动，实现的思路是：（因为涉及到页面其他比较繁琐的逻辑，这里就不贴代码了）

  • 主页面componentDidShow时，请求一下结算页面数据，储存在store中
  • 子页面根据相印操作再次拉取数据，并更新store

支付模块

支付模块是结算页面的重中之重。toplife小程序支付的流程如下。

  1. 提交订单（后端返回订单号等信息）
  2. 调取内部支付系统取得微信支付所需参数
  3. 调取微信小程序 requestPayment API

这里的坑就是 requestPayment 需要支付相关的userinfo：iv和encryptedData，而这些信息需要通过wx.getUserInfo接口拿到，但由于接口调整的原因，在用户未授权下，调取该接口不再弹出浮层提示用户授权，而是需要用户手动去点击小程序中open-type为getUserInfo的button组件以进行授权。所以这就需要在全局维护一个是否授权字段，若未授权，在点击支付时再弹出自己模拟的模态框引导用户授权。效果如下：
